# 提交前检查清单

在你提交新的学校适配或修复前，请尽量过一遍这张表。

---

## A. 网页与登录

- [ ] 能进入正确的登录页
- [ ] 登录后能稳定保持会话
- [ ] 如果有 CAS / SSO / 跳转，已经验证完整路径
- [ ] 如果有验证码 / 滑块，至少写明当前限制或处理方式

---

## B. 课程数据

- [ ] 课程名正确
- [ ] 教师正确
- [ ] 地点正确
- [ ] 星期正确
- [ ] 起止节次正确
- [ ] 周次正确
- [ ] 单双周没有反
- [ ] 自定义周 / 跳周没有明显遗漏
- [ ] 连堂课没有错误拆分或错误合并

---

## C. 时间模板与学期配置（如果脚本输出了）

- [ ] 节次编号正确
- [ ] 开始时间 / 结束时间正确
- [ ] 学期开始日期合理
- [ ] 总周数合理

---

## D. 仓库文件

- [ ] `adapters.yaml` 字段完整
- [ ] `import_url` 指向正确登录页或合理留空
- [ ] `maintainer` 可识别
- [ ] `description` 能说明适配用途和限制
- [ ] `asset_js_path` 指向正确文件
- [ ] 新学校已在 `index/root_index.yaml` 登记

---

## E. 测试流程

- [ ] 浏览器扩展 Alpha 阶段已跑通
- [ ] 轻屿开发版 / 自定义仓库 Beta 阶段已测试
- [ ] 导入到 App 后和学校原系统基本一致
- [ ] 没有只在单一页面偶然成功、刷新后就失效

---

## F. 提交卫生

- [ ] 没提交账号、Cookie、Token、抓包隐私数据
- [ ] 没提交 `.omx/`、编辑器缓存、本地环境文件
- [ ] 没把临时测试脚本误当正式脚本提交
- [ ] 若引用上游脚本，已说明来源与差异
- [ ] 若新增轻屿专用行为，已同步更新文档
