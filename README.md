 # qingyu_warehouse

  本仓库用于 **轻屿课表（Qingyu）** 的教务适配脚本管理、索引维护与导入测试。

  它主要服务于：
  - 教务系统适配脚本开发
  - 适配资源仓索引维护
  - 轻屿课表开发版 / 测试版的导入验证

  ---

  ## 与上游仓库的关系

  本仓库 fork 自：

  - [XingHeYuZhuan/shiguang_warehouse](https://github.com/XingHeYuZhuan/shiguang_warehouse)

  我们的目标不是简单镜像，而是：

  1. **尽量兼容上游仓库结构与适配协议**
  2. **为轻屿课表提供可独立维护的适配仓**
  3. **在必要时进行轻屿侧二次开发与扩展**

  如无特殊说明，本仓库优先沿用上游的资源组织方式与适配思路。

  ---

  ## 仓库定位

  请注意：

  - 本仓库是 **适配资源仓**
  - 不是轻屿课表主应用仓库
  - 不负责 App UI、课表本地管理、提醒逻辑等主应用能力
  - 主要负责：
    - 学校/工具索引
    - 适配器元数据
    - 教务导入脚本

  ---

  ## 兼容策略

  ### 当前兼容目标
  - `index/root_index.yaml`
  - `resources/<folder>/adapters.yaml`
  - `resources/<folder>/*.js`

  ### 原则
  - 优先兼容上游已存在的数据结构
  - 尽量让上游适配脚本可以迁移到本仓库
  - 如需增加轻屿课表专用扩展，将尽量保持向后兼容

  ### 不保证
  - 不保证所有上游脚本无需修改即可直接用于轻屿课表
  - 不保证本仓库的扩展字段能被上游软件识别
  - 实际行为以轻屿课表当前版本实现为准

  ---

  ## 仓库结构

  ```text
  index/
    root_index.yaml        # 学校/工具总索引

  resources/
    <resource_folder>/
      adapters.yaml        # 当前学校/工具的适配器配置
      xxx.js               # 具体适配脚本

  scripts/
    ...                    # 索引或构建辅助脚本

  ———

  ## 开发流程

  ### 1. 新增学校/工具

  先在 index/root_index.yaml 中登记条目。

  ### 2. 创建资源目录

  在 resources/ 下建立对应目录，并添加：

  - adapters.yaml
  - 对应适配脚本 *.js

  ### 3. 编写适配脚本

  脚本应完成：

  - 登录页识别 / 登录流程处理
  - 课程数据抓取
  - 数据解析与结构化输出

  ### 4. 在轻屿课表中测试

  建议使用轻屿课表开发版，在 App 内配置自定义仓库地址，拉取本仓库资源进行实际测试。

  ### 5. 验证通过后再提交

  确保：

  - 索引正确
  - 元数据完整
  - 脚本能正常导入
  - 不提交纯测试占位脚本

  ———

  ## 测试建议

  建议至少验证以下内容：

  - 能否正确进入教务系统登录页
  - 登录后是否能拿到课表数据
  - 是否能正确解析：
      - 课程名
      - 教师
      - 地点
      - 周次
      - 单双周
      - 节次 / 时间
  - 导入后的课表是否与原系统一致

  ———

  ## 提交建议
