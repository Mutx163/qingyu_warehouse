# qingyu_warehouse

轻屿课表（Qingyu）使用的 **教务适配资源仓库**。

本仓库主要用于：

- 教务系统适配脚本管理
- 学校 / 工具索引维护
- 轻屿课表开发版 / 测试版导入验证
- 与上游适配仓协议兼容下的独立维护与二次开发

---

## 与上游仓库的关系

本仓库 fork 自：

- [XingHeYuZhuan/shiguang_warehouse](https://github.com/XingHeYuZhuan/shiguang_warehouse)

本仓库不是简单镜像，而是在尽量兼容上游结构与适配协议的前提下，服务于 **轻屿课表** 的导入系统建设。

我们的目标是：

1. **尽量兼容上游仓库结构与适配协议**
2. **为轻屿课表提供可独立维护的适配仓**
3. **在必要时进行轻屿侧二次开发与扩展**

如无特殊说明，本仓库优先沿用上游的资源组织方式、索引结构与适配思路。

---

## 仓库定位

请注意：

- 本仓库是 **适配资源仓**，不是轻屿课表主应用仓库
- 不负责 App UI、课表本地管理、提醒逻辑等主应用能力
- 主要负责：
  - 学校 / 工具入口索引
  - 适配器元数据
  - 教务导入脚本
  - 适配调试与验证

如果你想开发或修复“某个学校怎么导入”，这个仓库就是对应的位置。

---

## 兼容策略

### 当前兼容目标

本仓库当前尽量兼容以下上游约定：

- `index/root_index.yaml`
- `resources/<folder>/adapters.yaml`
- `resources/<folder>/*.js`

### 兼容原则

- 优先兼容上游已存在的数据结构
- 尽量让上游适配脚本可以迁移到本仓库
- 如需增加轻屿课表专用扩展，将尽量保持向后兼容

### 不保证事项

- 不保证所有上游脚本无需修改即可直接用于轻屿课表
- 不保证本仓库的扩展字段能被上游软件识别
- 实际行为以轻屿课表当前版本实现为准

---

## 仓库结构

```text
index/
  root_index.yaml        # 学校 / 工具总索引

resources/
  <resource_folder>/
    adapters.yaml        # 当前学校 / 工具的适配器配置
    xxx.js               # 具体适配脚本

scripts/
  ...                    # 索引或构建辅助脚本

proto/
  ...                    # 协议与结构定义
```

## 开发流程

### 1. 新增学校 / 工具

先在 `index/root_index.yaml` 中登记条目。

### 2. 创建资源目录

在 `resources/` 下建立对应目录，并添加：

- `adapters.yaml`
- 对应适配脚本 `*.js`

### 3. 编写适配脚本

脚本应完成以下工作：

- 登录页识别 / 登录流程处理
- 课程数据抓取
- 数据解析与结构化输出

### 4. 在轻屿课表中测试

建议使用轻屿课表开发版，在 App 内配置自定义仓库地址，拉取本仓库资源进行实际测试。

### 5. 验证通过后再提交

提交前请至少确认：

- 索引正确
- 元数据完整
- 脚本能正常导入
- 不提交纯测试占位脚本

---

## 测试建议

建议至少验证以下内容：

- 能否正确进入教务系统登录页
- 登录后是否能拿到课表数据
- 是否能正确解析：
  - 课程名
  - 教师
  - 地点
  - 周次
  - 单双周
  - 节次 / 时间
- 导入后的课表是否与原系统一致

如果脚本涉及复杂登录流程（CAS / SSO / 验证码 / 重定向），请额外记录测试路径与失败场景。

## 提交建议

建议在提交说明中写清：

- 适配学校 / 系统名称
- 适配范围
- 是否新增字段
- 是否与上游兼容
- 测试环境与验证结果

如果是基于上游已有脚本修改，也建议注明来源与差异点，方便后续同步与维护。

## 注意事项

- 尽量保留上游兼容性，避免无必要破坏目录结构
- 如引用或修改上游已有适配脚本，请保留原始贡献信息
- 对远程脚本执行、登录凭据处理、数据抓取行为要保持谨慎
- 不要在主分支长期保留临时测试脚本
- 如扩展轻屿课表专用字段，请同步记录用途与兼容策略
