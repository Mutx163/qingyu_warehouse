# Contributing to qingyu_warehouse

感谢你为 `qingyu_warehouse` 贡献学校适配或脚本修复。

本仓库是 **轻屿课表** 的教务适配资源仓，fork 自 `XingHeYuZhuan/shiguang_warehouse`，目标是在尽量兼容上游协议的前提下进行独立维护与二次开发。

---

## 贡献前请先确认

你准备提交的是以下哪一类内容：

1. **新增学校 / 工具适配**
2. **修复已有学校脚本**
3. **补充或修正索引信息**
4. **协议兼容 / 仓库结构优化**
5. **轻屿课表专用扩展**

如果改动会影响上游兼容性，请在提交说明中明确写出。

---

## 基本原则

- 优先保持与上游仓库结构兼容
- 一个提交尽量只做一类事情
- 不提交纯测试垃圾文件、临时脚本或抓包产物
- 不把账号、Cookie、Token、验证码样本等敏感信息提交进仓库
- 如果修改基于上游已有脚本，请注明来源与差异点

---

## 目录约定

通常你会改这些位置：

- `index/root_index.yaml`
- `resources/<folder>/adapters.yaml`
- `resources/<folder>/*.js`

请尽量遵守现有目录和命名方式，避免引入无必要的新层级。

---

## 推荐开发流程

### 1. 先更新索引
如果是新增学校 / 工具，先在 `index/root_index.yaml` 中登记。

### 2. 创建或修改资源目录
在 `resources/<folder>/` 中维护：

- `adapters.yaml`
- 对应 JS 适配脚本

### 3. 本地或开发版验证
建议至少验证：

- 能否进入正确登录页
- 登录后能否抓到课表数据
- 课程字段是否解析正确
- 周次 / 单双周 / 节次时间是否正确
- 导入到轻屿课表后是否和原系统一致

### 4. 写清提交说明
提交说明至少建议包含：

- 学校 / 系统名称
- 改动类型（新增 / 修复 / 兼容）
- 是否影响上游兼容
- 测试结果

---

## Pull Request 建议模板

建议在 PR 描述中回答这些问题：

- 适配的是哪所学校 / 哪类系统？
- 这次改动解决了什么问题？
- 是否新增了字段或扩展行为？
- 是否在轻屿课表开发版中验证过？
- 是否会影响已有学校 / 上游兼容？

---

## 关于轻屿课表专用扩展

如果你要增加轻屿课表专用字段或行为：

- 请尽量保持向后兼容
- 请在 PR 中单独说明用途
- 不要默认假设上游仓库或其他客户端一定支持

---

## 不建议提交的内容

- 临时抓包文件
- 包含账号密码 / Cookie / Token 的样本
- 与单个设备强绑定的调试文件
- 未整理的测试占位脚本
- 与本仓库目标无关的主应用代码
